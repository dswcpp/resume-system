### 项目一：带电检测手持移动终端（T95）


| 项目信息 | 详情               |
| ---- | ---------------- |
| 时间周期 | 2023.04 - 至今     |
| 所属公司 | 南京华乘电气技术有限公司     |
| 项目规模 | 开发团队12人，用户数量500+ |
| 我的角色 | 中级开发工程师，核心模块负责人  |


**项目背景**：

该项目是一款多功能电力设备带电检测终端，集成了红外测温、超高频（UHF）局部放电检测、暂态地电压（TEV）检测、声发射（AE）检测等多种检测技术。系统用于电力设备状态监测和故障诊断，帮助电力工作人员及时发现设备潜在故障，预防电气事故发生。

**技术栈**：

Qt/C++、信号槽机制、QThread多线程、状态机模式、生产者-消费者模式、SPSC无锁队列、USB/串口通信、PRPS模式识别、FFT频谱分析

**负责内容**：

1. **多检测技术集成框架设计**
  - 设计统一的检测模块接口规范，定义数据采集、处理、展示的标准流程
  - 实现模块热插拔机制，支持检测模块的动态加载与卸载
2. **TEV暂态地电压检测模块开发**
  - 实现高速脉冲采集（采样率1MHz），波形数据缓冲与处理
  - 开发PRPS（相位分辨脉冲序列）图谱生成算法
  - 实现放电类型识别与严重程度评估
3. **红外成像模块集成**
  - 设计基于状态机的设备连接管理机制（断开→连接中→已连接→异常）
  - 实现伪彩图显示、温度标定、最高温追踪功能
  - 2周内完成从SDK调研到功能上线
4. **UHF超高频数据处理优化**
  - 优化数据采集流程，提升采样率与处理效率
  - 实现频谱分析与特征提取算法
5. **实时数据管线重构**
  - 采用生产者-消费者模式解耦数据采集与UI刷新
  - 使用SPSC无锁队列降低线程同步开销
  - 实现控频刷新策略，避免UI过度刷新

**项目难点与解决方案**：


| 难点              | 解决方案                                         |
| --------------- | -------------------------------------------- |
| 多检测技术数据格式和采样率各异 | 设计统一数据接口与转换层，实现异步通信架构，通过时间戳对齐解决不同采样率数据同步问题   |
| 大数据量实时处理性能瓶颈    | 生产者-消费者模式 + SPSC无锁队列 + 控频刷新，UI延迟从200ms降至50ms |
| 设备连接不稳定         | 状态机管理连接状态，实现自动重连、超时检测、异常恢复机制                 |
| 跨平台兼容性问题        | 抽象平台相关代码为独立模块，开发可移植组件库                       |


**项目成果**：

- 模块化设计使新检测技术集成周期从3个月缩短至约50天
- 系统检测准确率超过92%，获得多家电力企业客户采购
