# 你的期望薪资是多少？

## 回答框架

薪资谈判的核心原则：**用成果支撑数字，给范围而非死数，表达灵活性**。结构是：报出期望范围 → 用 2-3 个核心成果支撑 → 表达看重整体机会。

## 我的实战经历

### 推荐回答（自信简洁版）

"期望在 25K 左右。这个期望基于三点：

**第一，解决问题的能力。** T95 项目 UI 卡顿问题，我用无锁队列把延迟从 200ms 降到 50ms，CPU 占用降了 40%。这种性能优化能力在市场上是有溢价的。

**第二，架构设计能力。** 我设计的模块化集成框架，让新检测技术的接入周期从 3 个月缩短到 50 天，直接提升了团队交付效率。

**第三，综合经验。** 5 年 C++ 经验，从 PCB 喷墨设备到银行终端再到电力检测，串口、RS485、Modbus、MQTT 我都实际用过，能快速对接各种硬件。还有带新人的经验。

当然具体可以根据岗位职责和考核标准来协商，我更看重整体的成长机会和技术平台。"

## 延伸思考

### 常见追问应对

**"你现在多少钱？"**
"我目前的薪资是 [如实说]。25K 的期望是基于我的能力成长和市场行情，而不是简单在现有基础上加多少。"

**"25K 超出我们预算了"**
"我理解。薪资只是一部分，我也会综合考虑技术平台、成长空间、团队氛围等因素。如果整体机会足够好，薪资方面我们可以进一步商量。"

**"能不能给个底线？"**
"我觉得与其纠结底线，不如聊聊岗位的具体职责和期望。如果岗位的挑战和成长空间跟我预期一致，薪资方面我相信我们能找到双方都满意的方案。"

### 薪资谈判的禁忌

| 避免 | 正确做法 |
|------|----------|
| "我觉得我值这个价" | "基于我的成果和市场行情..." |
| "市场行情就是这样" | "我能带来这些价值..." |
| "其实可以少一点" | "可以根据职责协商" |
| "我上家就是这个数" | "我的能力体现在..." |
| 第一轮面试就谈钱 | 等对方主动提起或 HR 环节 |

### 如何判断自己的市场价值

- 招聘网站同类岗位薪资范围
- 同行朋友的实际薪资水平
- 猎头给出的参考范围
- 自己的不可替代性和核心竞争力

## 面试表达建议

**自信但不傲慢**：数据说话比主观评价更有说服力。"我值 25K" 不如 "延迟从 200ms 降到 50ms"。

**留余地**：说"左右"而非精确数字，给双方协商空间。

**最后表达灵活**：一定要加一句"可以根据职责协商"或"看重整体机会"，避免给人"只看钱"的印象。

**时机把握**：最好在了解岗位详情后再谈薪资。如果面试一开始就问，可以说"我想先了解一下岗位的具体职责和要求，这样我能给出更合理的预期。"
