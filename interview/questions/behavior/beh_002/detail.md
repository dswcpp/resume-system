# 为什么想离开现在的公司？

## 回答框架

离职原因类问题的核心原则：**不抱怨、不贬低、聚焦成长**。把"逃离旧环境"转化为"追求新机会"。结构是：当前状态客观描述 → 个人发展诉求 → 与目标公司的匹配。

## 我的实战经历

### 推荐回答

"主要是寻求更大的技术挑战和成长空间。

T95 项目是我目前负责的核心项目，我在里面做了不少有成就感的事——无锁队列优化、模块化框架设计、带新人。但项目现在已经进入稳定维护期，核心架构和模块都已落地，日常主要是修复零星 Bug 和做小幅优化，技术上的挑战在减少。

我希望能接触更大规模、更复杂的系统，进一步提升架构设计能力。比如分布式系统、更高并发的场景，或者技术栈更广的团队。

了解到贵司在 [具体方向/产品] 上有很深的技术积累，跟我的发展规划很契合，所以想争取这个机会。"

### 根据不同情况的变体

**如果被追问"是不是钱不够"：**
"薪资是一方面，但不是主要原因。我更看重的是技术成长空间和团队的技术氛围。如果只是为了涨薪，跳去一个没有技术挑战的岗位对我没意义。"

**如果被追问"跟领导/同事有矛盾吗"：**
"没有，团队协作一直不错。领导也很支持我的技术方案，比如无锁队列重构就是领导给了我充分的信任才能推进的。主要还是项目阶段的原因。"

## 延伸思考

### 绝对不能说的话

| 避免说法 | 替代说法 |
|----------|----------|
| "领导不好/管理混乱" | "寻求更好的技术成长环境" |
| "工资太低" | "希望能力和价值更匹配" |
| "太累了/加班多" | "希望更好的工作生活平衡" |
| "公司要倒闭了" | "行业/方向调整" |
| "同事不好相处" | "希望在更大的技术团队中学习" |

### 面试官在考察什么？

1. **稳定性**：你会不会很快又跳槽？→ 强调是项目阶段原因而非个人习惯
2. **人品**：你会不会背后说前公司坏话？→ 对前公司保持尊重
3. **驱动力**：你的核心诉求是什么？→ 技术成长，不是逃避
4. **匹配度**：你想要的我们能提供吗？→ 提前了解目标公司

### 补充：如何表达对前公司的感谢

"在华乘两年多，我成长了很多。从普通开发到核心模块负责人，从写代码到设计架构、带团队，这些经验对我帮助很大。只是现在项目进入维护期，我希望在新的环境继续保持这种成长节奏。"

这样说既诚实又体面，面试官会觉得你是个有感恩心的人。

## 面试表达建议

**语气平和真诚**：不要表现出对前公司的不满，即使确实有不满。面试官会想"他今天说前公司坏话，明天也会说我们的"。

**控制时间**：30 秒内回答完，不要展开太多。这个问题不是加分项，答得越多越容易出错。

**过渡到正面**：回答完离职原因后，主动过渡到"为什么选择贵司"，把话题从负面（离开）转向正面（加入）。
