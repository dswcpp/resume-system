# 你指导初级工程师有什么经验？

## 回答框架

指导新人类问题考察**团队贡献和领导力潜质**。用 STAR 结构讲一个具体案例，展示系统化的指导方法和可衡量的成长结果。核心是：有方法论、有耐心、有成果。

## 我的实战经历

**公司/项目：** 南京华乘电气 · T95 带电检测手持终端

**情境（S）：**
去年团队来了 2 名应届生，C++ 基础不错但缺乏工程经验——不熟悉代码规范、不了解多线程编程的坑、不会做性能分析。领导让我负责带他们，目标是 3 个月内能独立承担模块开发任务。

**任务（T）：**
把两个基础不错但零经验的应届生，在 3 个月内培养成能独立负责子模块的工程师。挑战在于：项目进度紧张不能让他们慢慢学，但也不能让他们只做简单任务失去成长空间。

**行动（A）：**

**方法一：渐进式任务拆解**

不是直接丢一个大模块让他们从零开始，而是按能力梯度分配任务：

| 阶段 | 时间 | 任务 | 目的 |
|------|------|------|------|
| 熟悉 | 第 1 周 | 读代码、修 3 个小 Bug | 理解项目结构 |
| 小功能 | 第 2-4 周 | 实现 UI 适配功能 | 熟悉开发流程 |
| 子模块 | 第 2 月 | 负责协议解析模块 | 独立设计和编码 |
| 独立 | 第 3 月 | 独立负责新模块 | 全流程验证 |

每完成一个阶段都有成就感，不会产生"我好菜什么都不会"的挫败感。

**方法二：每次提交必 Code Review**

这是最花时间但效果最好的方法。不只说"这里不对"，而是**解释为什么、怎么改更好**：

```
// 我的Review批注示例：
"这里用了裸指针 new，建议改成 unique_ptr。原因有三：
1. 当前代码如果中间 return 了会内存泄漏
2. unique_ptr 明确表达了所有权
3. 我们的编码规范要求避免裸 new
具体怎么改：auto p = std::make_unique<Module>();"
```

Review 中常见问题记录下来整理成 Checklist，包括：
- 内存管理（智能指针 vs 裸指针）
- 多线程安全（是否需要加锁、锁的粒度）
- 错误处理（返回值是否检查、异常是否处理）
- 日志规范（什么级别、什么内容）

**方法三：技术分享驱动学习**

组织过两次主题分享：
- "C++ 性能优化实战"：结合无锁队列案例讲 atomic、memory_order
- "状态机设计模式"：用红外模块连接管理做案例

让新人参加分享时做笔记，之后把理解整理成文档我来审核补充。准备分享的过程本身就是深度学习。

**方法四：鼓励文档习惯**

要求他们写开发笔记：遇到什么问题、怎么解决的、学到了什么。我定期审核补充。一方面帮助他们总结提炼，另一方面留下知识沉淀供后人参考。

**结果（R）：**

- 3 个月后两人都能**独立承担子模块开发**
- 6 个月后各自独立负责一个完整的检测模块
- Code Review 意见从每次 10+ 条降到 2-3 条
- 其中一位现在已经在带更新的新人了
- 沉淀了一套 Code Review Checklist 和新人培养路径文档，被团队持续使用

## 延伸思考

### 带新人最大的误区

1. **直接给答案**：应该引导他自己找到答案。"你觉得这里为什么会崩溃？加个断点看看那个指针是不是空的？"
2. **任务太难或太简单**：太难打击信心，太简单学不到东西。梯度很重要。
3. **只Review不教**：光说"这里不对"没用，要解释 why 和 how。
4. **不放手**：到了一定阶段要让他独立负责，犯点小错比永远在你保护下好。

### 如果新人学得慢怎么办？

1. 先判断是**能力问题**还是**态度问题**
2. 能力问题：调整任务难度、增加辅导频率
3. 态度问题：明确期望、设立里程碑、定期反馈
4. 如果确实不合适：及时向领导反馈，不要拖

### 带新人对我自己的帮助

- 教别人的过程加深了自己的理解（费曼学习法）
- 被新人问"为什么"时发现自己一些习以为常的做法其实没想透
- Review 别人代码的过程提升了自己的代码审查能力

## 面试表达建议

**用具体例子，不要泛泛而谈**："我每次提交都 Review，不只说哪里不对，还解释为什么和怎么改"——这比"我会认真带他们"有说服力得多。

**展示系统方法**：渐进式任务拆解 + 代码 Review + 技术分享 + 文档习惯——说明你不是随便带的，有方法论。

**用结果证明效果**："3 个月独立承担模块"、"Review 意见从 10 条降到 2 条"——量化的成长轨迹。

**收尾展示格局**："其中一位现在已经在带更新的新人了"——说明你的指导方法是可传承的。
