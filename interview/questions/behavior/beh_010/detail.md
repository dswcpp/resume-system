# 你有什么问题想问我们的？

## 回答框架

反问环节是面试的最后机会，考察你的**深度思考能力和对岗位的认真程度**。准备 4-5 个问题，根据面试节奏选 2-3 个问。优先问技术和成长相关的，避免薪资福利类（留给 HR 环节）。

## 我的实战经历

### 技术相关问题（选 1-2 个）

**问题 1："团队目前 C++ 主要用在什么场景？整体技术栈和架构是怎样的？"**

为什么问这个：了解你将要做什么，判断是否匹配。同时展示你关注技术架构全貌。

**问题 2："这个岗位主要负责哪个模块？目前最大的技术挑战是什么？"**

为什么问这个：了解具体工作内容和难度。如果面试官说了挑战，你可以接着聊你的相关经验——"我在 T95 项目中也遇到过类似的问题，当时是这样解决的……"

**问题 3："团队的代码规模大概多少？用什么构建系统？CI/CD 流程是什么样的？"**

为什么问这个：判断团队的工程化水平。成熟的团队有完善的构建和测试流程，这对你的成长很有帮助。

### 成长相关问题（选 1 个）

**问题 4："新人入职前三个月，最希望我重点补什么技能？"**

为什么问这个：展示你的主动性和学习意愿。面试官的回答也能帮你判断岗位需求是否匹配。

**问题 5："团队有 Code Review 机制吗？有技术分享或者内部培训吗？"**

为什么问这个：了解团队的技术文化。有 Code Review 和技术分享的团队通常技术氛围好、成长快。

### 团队文化问题（选 1 个）

**问题 6："团队的工作节奏是怎样的？一个典型的迭代周期大概多久？"**

为什么问这个：了解工作强度和开发模式（敏捷/瀑布）。

**问题 7："产品未来的发展方向是什么？"**

为什么问这个：了解公司前景和你可能的发展空间。

### 根据面试情况灵活调整

**如果面试聊得很愉快**：可以问更深入的技术问题
**如果面试中暴露了你的不足**：问"新人入职前三个月最希望补什么"来展示学习意愿
**如果面试时间很紧**：只问一个最核心的问题

## 延伸思考

### 绝对不要问的问题

| 不要问 | 原因 |
|--------|------|
| "加班多吗？" | 给人不积极的印象 |
| "薪资多少？" | 留给 HR 环节 |
| "你们公司做什么的？" | 说明你没做功课 |
| "面试结果什么时候出？" | 显得急切 |
| "我表现怎么样？" | 让面试官尴尬 |

### 高级技巧：把反问变成展示

当面试官回答你的问题时，你可以接着展示相关经验：

面试官："我们现在最大的挑战是多线程数据处理的性能问题。"
你："理解。我在 T95 项目中也处理过类似的问题——采集线程 1000Hz，UI 线程 30Hz，用 SPSC 无锁队列把延迟从 200ms 降到了 50ms。如果贵司的场景类似，我有信心能快速上手。"

这样反问环节不仅是提问，还是最后一次展示你的价值。

### 如果面试官说"没什么好问的，你还有问题吗？"

一定要问至少一个问题。说"没问题了"等于放弃了最后的机会，也可能让面试官觉得你不感兴趣。

准备一个**万能问题**作为保底：
"如果我有幸加入团队，您最希望我在前三个月做出什么样的贡献？"

这个问题展示了积极性、务实性，而且面试官的回答能帮你判断岗位是否合适。

## 面试表达建议

**提前准备，但不要背**：准备 4-5 个问题，面试时根据氛围选择。不要照着纸读。

**认真听面试官的回答**：不要问完就走神，面试官的回答可能包含重要信息，也是你展示互动能力的机会。

**控制时间**：每个问题聊 1-2 分钟就好，不要占用面试官太多时间。

**最后感谢**：反问结束后说一句"谢谢，跟您聊了很多收获很大"——礼貌得体的结尾留下好印象。
