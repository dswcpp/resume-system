# 段胜炜｜职业规划建议（2026 版）

> 目标：基于你当前履历（C++/Qt、设备通信、并发与实时数据管线、协议与状态机、现场稳定性与性能优化），形成一份可执行、可迭代的 2026 职业规划。
> 
> 建议的主线：**工业/设备端的高性能 C++（实时 + 通信 + 架构）**。这条线稀缺、议价能力强，且与你的项目经历高度匹配。

---

## 0. 一句话定位（建议写进简历/自我介绍）

你可以用以下 3 句之一作为对外定位（按“高级感”排序）：

1. **高性能 C++/Qt 工程师（实时数据 + 设备通信 + 系统稳定性）**：擅长把“数据采集—处理—显示—存储”链路做稳做快，能在现场复杂干扰下保证可靠性。
2. **工业终端/测控系统 C++ 工程师**：负责多传感/多协议接入、状态机解析与容错、数据流并发架构及性能调优。
3. **嵌入式应用/上位机 C++ 工程师（偏架构与性能）**：以模块化、可移植、可观测为导向推动迭代。

> 你当前写“中级”，但内容已经接近“高级岗位”的工作范畴；规划的核心是：**补齐高级岗位的证据链**（可量化指标 + 工程化 + 公开作品）。

## 1. 现状评估：优势、短板、机会

### 1.1 你的可迁移优势（强烈建议保留并强化）

- **实时数据链路**：你已经做过“生产者-消费者 + SPSC/无锁队列 + 控频刷新”，这属于高价值稀缺经验。
- **设备通信/协议可靠性**：USB/串口、RS485、Modbus、MQTT 等 + 状态机解析 + CRC/重试/轮询策略，面向工业现场很吃香。
- **Qt/C++ 工业 UI 与多线程**：工业软件/终端侧岗位的主力技能。
- **现场稳定性与性能优化**：能把“吞吐、延迟、故障率、CPU 占用”用数据讲清楚，是涨薪跳槽的关键。

### 1.2 常见短板（补齐后可显著提高上限）

- **Linux 工程化与性能工具链**：gdb/perf/valgrind、系统服务化、部署与可观测。
- **现代 C++ 深水区**：并发内存模型、锁策略、对象生命周期、性能陷阱、库设计。
- **工程体系**：CMake、单测（gtest）、CI、编码规范与静态分析、版本化接口。

### 1.3 机会方向（与你经历匹配且更“贵”）

- 工业自动化/测控/仪器仪表
- 电力/能源行业的软件平台化（边缘终端、综合在线监测、数据汇聚）
- 机器人/视觉/智能制造（你有 Halcon/OpenCV 背景）
- 边缘计算网关/工控安全（大量协议/通信 + 稳定性要求）

## 2. 目标设定：2026 年的三档目标

> 你当前简历里期望 22K–28K。建议把目标拆成三档，便于推进。

- **保守目标**（稳）：稳定在 **25K±**，岗位：高级 C++/Qt（偏工业终端）。
- **进取目标**（跳）：冲击 **30K–40K**，岗位：C++ 架构/性能/平台化（小团队或核心模块负责人）。
- **激进目标**（转型/产品化）：主业 + 副业/咨询/接单形成第二增长曲线，全年额外收入 **5–20 万**。

## 3. 方向选择：三条主线（建议选 1 条为主，另 1 条为备）

### A. 高级 C++/Qt（架构 + 性能 + 可维护）【推荐主线】
适合：你现在的经历延续性最强。

- 核心卖点：实时数据管线、协议可靠性、模块化架构、性能优化。
- 目标岗位关键词：Qt、C++17、并发、数据可视化、采集与解析、工业软件。

### B. 工业边缘计算/测控平台（通信 + 实时 + 部署）
适合：想把影响范围做大（从终端到系统）。

- 需要补：Linux 服务化、容器/部署、指标与日志体系。

### C. 终端侧负责人（带 1–3 人，小团队交付）
适合：你愿意承担“方案 + 交付 + 质量”的责任。

- 核心卖点：把项目跑起来、按期交付、现场问题闭环。
- 需要补：需求拆解、排期、跨团队协作、质量体系。

## 4. 12 个月执行计划（按季度）

### Q1（第 1–3 个月）：证据链搭建（最关键）

**目标**：让“高级能力”可见、可验证。

- 建 1 个公开项目（GitHub/Gitee）：
  - 主题建议：**“实时数据采集模拟器 + SPSC 队列 + Qt 波形/PRPS 可视化 + 性能基准”**
  - 必备内容：
    - 压测脚本（吞吐/延迟/CPU/内存）
    - 日志/指标（最简也要有）
    - README 写清楚架构、线程模型、瓶颈与优化
- 简历升级：每个项目至少补 2 个量化指标（吞吐、延迟、故障率、CPU 占用、内存峰值、集成周期等）。

### Q2（第 4–6 个月）：工程化与 Linux 能力补齐

- CMake：能从 0 搭出可复用工程模板（lib + app + tests）。
- 单测：至少掌握 gtest + 基本覆盖。
- 性能工具链：
  - Windows：VS profiler/ETW（如可用）
  - Linux：gdb + perf + valgrind（重点）
- 输出 2 篇技术复盘（可公开/可内部）：
  - 一篇性能定位（数据—假设—实验—结论）
  - 一篇协议可靠性（状态机 + 容错策略）

### Q3（第 7–9 个月）：面试准备 + 试投 + 议价能力提升

- 面试“故事库”（至少 6 个故事）：
  1) 一个性能瓶颈如何定位与验证
  2) 一个协议不稳如何提高可靠性
  3) 一个架构如何抽象成可插拔模块
  4) 一次线上/现场故障如何复盘与沉淀
  5) 如何带新人/做 code review
  6) 与硬件/算法/测试协作的案例
- 试投 10–20 家目标公司，拿反馈迭代简历与自我介绍。

### Q4（第 10–12 个月）：跳槽/晋升落地或副业产品化

- 若主业跳槽：
  - 目标薪资与岗位要明确（例如 30K+ 的“高级/资深”而非“中级”）
  - 核心谈判点：你能“把系统跑稳跑快”，且有证据
- 若主业稳定：
  - 副业产品化（见第 7 章）或内部晋升路径

## 6. 求职策略（公司类型与岗位筛选）

### 6.1 推荐优先级

1) **工业软件/仪器仪表/测控**（稳定 + 技术硬 + 议价空间）
2) **机器人/视觉/制造**（对并发与设备更友好）
3) **边缘计算网关/工控安全**（协议多、稳定性要求高）

### 6.2 简历与面试的“抓手”

你简历里已经有很好的数字（如 15MB/s、92%、集成周期缩短等）。建议进一步统一口径：

- 背景：为什么难（现场干扰/多协议/高吞吐/实时 UI）
- 动作：你怎么做（架构/线程模型/协议策略/压测）
- 结果：用指标说话（吞吐、延迟、CPU、故障率、交付周期）
- 沉淀：形成模板/组件/工具（提高复用）

## 7. 2026 的“挣钱”建议（主业 + 副业）

> 你最适合的副业不是泛泛的 AI 应用，而是**“工业/设备通信自动化 + 交付”**，客单价高、客户明确。

### 7.1 副业方向 1：设备通信/数据采集“快速交付包”

可卖：
- 设备接入（串口/RS485/Modbus/MQTT）+ 数据落库 + 简单看板
- 故障诊断工具（日志/回放/协议抓包/告警）

收费模型：
- 搭建费 3k–20k（按复杂度）
- 运维费 500–3000/月（现金流）

### 7.2 副业方向 2：Qt 工业 UI 性能优化/疑难排查

- 给企业做“卡顿/内存泄漏/线程死锁”诊断
- 按天收费（1k–3k/天）或按结果包干

### 7.3 副业方向 3：把你的工程模板产品化

- “Qt + CMake + gtest + 日志 + SPSC 队列 + 仿真数据源”开源/半开源
- 可卖：定制、培训、企业版支持
