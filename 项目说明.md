# 个人简历系统

## 系统主旨

个人简历系统旨在帮助求职者创建、管理和优化专业简历。该系统采用模块化设计，通过结构化的流程和专业的优化建议，使用户能够制作出真实、具体、专业且有竞争力的个人简历。

本系统不依赖前端界面和数据库，而是通过一系列markdown文档和规范化流程，帮助用户完成从信息收集到简历优化的全过程。

## 快速访问

### 本地访问方式（推荐）

1. 下载或克隆本仓库到本地
2. 直接在浏览器中打开 `简历.html` 文件即可查看精美简历
3. 支持直接打印或导出为PDF格式

### 在线访问方式 - 使用CDN加速（推荐）

通过jsDelivr CDN加速访问简历，速度快且稳定：

```
https://cdn.jsdelivr.net/gh/[你的GitHub用户名]/resume-system@main/简历.html
```

### 通过GitHub Pages访问

```
https://[你的GitHub用户名].github.io/resume-system/简历.html
```

需要先在GitHub仓库 Settings → Pages 中开启服务。

## 系统规划

### 文件结构

- `项目说明.md` - 系统说明文档（本文件）
- `系统手册.md` - 系统详细使用手册
- `简历模板.md` - 简历信息收集模板（⚠️ 含个人隐私信息）
- `简历草稿.md` - 简历优化稿
- `简历.html` - 精美的HTML格式简历（可直接在浏览器打开）
- `面试准备指南.md` - 面试准备总纲
- `职业规划建议-2026.md` - 2026年职业规划建议
- `doc/` - 技术文档库（6个核心技术文档 + 文档索引）
- `doc/interview_prep/` - 面试准备专区（27份面试准备材料）
- `examples/` - 代码示例

### 工作流程

1. **信息收集阶段**
- 完整填写 `简历模板.md` 文件
   - 确保所有关键信息真实、具体、完整

2. **内容优化阶段**
- 基于模板信息生成初稿 `简历草稿.md`
   - 按照专业简历标准优化内容
   - 确保每个描述具体、量化、突出成果

3. **格式调整阶段**
   - 创建最终版本 `resume_final.md`
   - 调整排版、格式、重点突出
   - 确保整体简洁专业

4. **导出与应用阶段**
   - 根据需要导出为PDF或生成HTML网页版本
   - 针对不同岗位可创建不同版本

## 核心原则

- **真实性** - 所有内容必须真实，不编造、不夸大
- **具体性** - 用数据和事实说话，避免模糊表述
- **相关性** - 内容与目标职位高度匹配
- **专业性** - 使用行业术语和规范表达
- **简洁性** - 表达精炼，突出重点

## 使用方法

1. 阅读 `系统手册.md` 了解系统详细使用说明
2. 按照模板填写 `简历模板.md` 文件
3. 根据优化建议完善 `简历草稿.md` 内容
4. 导出为PDF或使用 `简历.html` 在线展示
5. 利用 `doc/interview_prep/` 目录的材料准备面试

## 在线访问配置指南

### 配置GitHub Pages（免费托管）

1. 前往GitHub仓库 → Settings → Pages
2. Source选择"Deploy from a branch"
3. Branch选择"main"和"/(root)"，然后点击Save
4. 稍等片刻，简历即可通过 `https://[用户名].github.io/resume-system/简历.html` 访问

### 使用jsDelivr CDN加速（推荐）

无需额外配置，只需访问：
```
https://cdn.jsdelivr.net/gh/[用户名]/resume-system@main/简历.html
```

### 隐私提醒

⚠️ 如果将本仓库公开，请注意 `简历模板.md` 等文件包含个人隐私信息（手机号、邮箱、住址等）。建议：
- 公开前将敏感信息替换为占位符
- 或使用私有仓库
